# 004 不要な Agent Skill を発火させない

## 主題

最初の節約は、情報を短くすることではなく、不要な Skill を読ませないこと。

## 重要キーワード

- 不要な発火
- 発火条件
- 入口情報
- 参照先
- 文脈が増える
- 判断の焦点
- 距離感

## 重くなる流れ

```text
関係の薄い場面で Skill 発火
  ↓
SKILL.md が入る
  ↓
参照先も候補になる
  ↓
文脈が重くなる
```

## 軽くする考え方

- 必要なときだけ呼ぶ
- 発火範囲を設計する
- 反応のよさと軽さを両立する
- Skill 数が増えるほど重要

## 吹き出し案

「呼ばなくていい Skill は、そっと待っていてもらいます…」
