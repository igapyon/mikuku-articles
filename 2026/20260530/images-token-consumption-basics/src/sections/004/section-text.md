# 004 入力トークンと出力トークン

## 主題

トークン消費量は、入力側と出力側の両方で考える。

## キーワード

- 入力トークン
- 出力トークン
- 質問
- 会話履歴
- 添付ファイル
- 回答
- コード

## 図解の流れ

```text
質問・履歴・資料・ファイル
  ↓
入力トークン
  ↓
AIモデル
  ↓
出力トークン
  ↓
回答・説明・コード
```

## 注意点

- Markdown やテキストファイルは中身が入力になりやすい
- PDF、Word、Excel は抽出されたテキストや構造が入力になりうる
- 画像や音声も、対応モデルでは利用量と無関係とは限らない
- 短い質問と短い回答は軽くなりやすい
- 長い資料と長い出力は重くなりやすい

## みくく吹き出し案

「入力と出力、どちらでもトークンは増えるのです…」
