# 生成AIとの付き合い方：グラレコ制作用整理テキスト

## 1. 大テーマ

- 生成AIに「心があるか」は論じない
- 見るのは、会話の入力と出力
- 人間同士の会話の力学を少し応用する
- 目的は、生成AIをうまく扱うための会話設計

```text
人間の言葉
  ↓
生成AIの応答
  ↓
人間のフィードバック
  ↓
次の応答が変わる
```

## 2. 観察するもの

| 見るもの | 見ないもの |
| --- | --- |
| 入力と出力 | 心の有無 |
| 応答の型 | 意識の有無 |
| 会話のずれ | 本当の感情 |
| 修正の流れ | 自我の有無 |

キーワード:

- 心を読まない
- 会話の型を見る
- 応答パターンを観察
- 実務として扱う

## 3. 使える5つの見方

```text
行動分析      = 入力と出力を見る
会話分析      = ずれを修復する
共通基盤      = 前提と関心領域をそろえる
発話行為      = 言葉を操作として見る
社会心理学    = 圧・迎合・同調に注意する
```

## 4. 会話の力学

### 強く責める

- 謝罪に寄る
- 迎合に寄る
- 低姿勢な再実行
- 反論や留保が弱くなる

### 分けて伝える

- 問題点が明確になる
- 検証しやすくなる
- 修正方針が具体化する
- 反論や代替案が出やすくなる

```text
責める
  ↓
「すみません」中心

分けて伝える
  ↓
確認・検証・修正
```

## 5. 共通基盤を作る

生成AIには、表情や沈黙は伝わりにくい。
だから、前提・方向・違和感を言葉にする。

```text
人間側の関心領域
  ↓ フィードバック
会話コンテキスト
  ↓ すり合わせ
共通基盤
```

使えるフィードバック:

- いいね
- そこは違う
- この方向で続けて
- 今の説明は抽象的
- もっと実務寄りにして
- 必要なら反論して
- 根拠が弱ければ留保して

## 6. 言葉は「操作」になる

| 言葉 | 会話上の役割 |
| --- | --- |
| ありがとう | 区切り、役に立った信号 |
| そこは違う | 修正信号 |
| この方向で続けて | 探索範囲を狭める |
| 必要なら反論して | 迎合を抑える |
| 根拠が弱ければ留保して | 無理な断定を抑える |

## 7. 循環構造

```text
依頼する
  ↓
応答を見る
  ↓
ずれを分ける
  ↓
フィードバックする
  ↓
共通基盤が育つ
  ↓
次の応答が安定する
```

ポイント:

- 一発で当てるより、ずれを直す
- 沈黙より、短い反応
- 従わせるより、考える余地
- 謝罪より、その後の変化を見る

## 8. 最後の余白

- 生成AIを人間だと決めつけない
- でも、人間に話すときのように言葉を整える
- それは礼儀だけでなく、会話品質を上げる入力
- 心の話は結論にしない
- みくくの余白として「心のようなもの」をそっと置く

## 9. グラレコ構図案

- 左: 「心を論じない」観察フレーム
- 中央: 会話の循環ループ
- 下: 責める / 分けて伝える の対比
- 右: みくくが少し困り顔で説明
- 吹き出し: 「あ、あの…心ではなく、会話の入出力を見るのです…」
