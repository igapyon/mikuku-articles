# はじめに

## 主題

設計書ゼロの既存システムを、生成AIでいきなり完成設計書にするのではなく、ソフトウェア考古学のように痕跡から仕様メモを育てる。

## 図解キーワード

- 設計書ゼロ
- 既存システム
- ソースコードは事実を持つ
- 業務上の意味は別に探す
- 痕跡を拾う
- 仕様メモへ蒸留

## 流れ

```text
ソースコード / DB / 設定 / ログ
+ Excel / 運用メモ / 人の記憶
  ↓
仮説
  ↓
有識者レビュー
  ↓
使える仕様メモ
```

## 対比

| 単純な自動生成 | ソフトウェア考古学 |
| --- | --- |
| コードをそのまま文書化 | 痕跡を読み解く |
| 説明文は作れる | 理解の順番を作る |
| 業務意味が抜ける | 人の知識で照合する |
