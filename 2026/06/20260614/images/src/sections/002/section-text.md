# いきなり生成するのではなく、発掘する

## 主題

ソースコード全体を AI agent に一気に読ませるのは重すぎる。まず人間が理解しやすい見取り図を作る。

## 図解キーワード

- 一気読みは重い
- トークン消費
- 時間がかかる
- 出力が期待と違う
- 情報量と理解しやすさは別
- 見取り図
- 最初の遺構

## 対比

| いきなり全部 | まず発掘 |
| --- | --- |
| 全ソースを背負う | 見える入口から読む |
| 情報過多 | 小さな見取り図 |
| 完成版を期待 | 仮説から始める |

## 図の流れ

```text
巨大なソースコード
  ↓ そのままでは重い
見えている遺構
  ↓
小さな地図
  ↓
次に読む場所が決まる
```

## 吹き出し案

「情報量が多いことと、理解しやすいことは別なのです…」
