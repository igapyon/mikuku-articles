# 機能一覧は、少し深い地層にある

## 主題

機能はクラス名やメソッド名としてそのまま出ないことがある。画面、バッチ、帳票、API の一覧が育つと、機能一覧の輪郭が見える。

## なぜ難しいか

- 1画面で複数業務をする
- 複数画面をまたぐ業務機能がある
- 人間の操作や判断が入る
- システムに現れない人間系機能がある

## 流れ

```text
画面 / バッチ / 帳票 / API
  ↓ 材料が増える
機能の輪郭
  ↓
不完全な機能一覧
  ↓
ヒアリングしやすい叩き台
```

## 注意点

- 廃止機能は大きなリスク
- 補完は仕様確定ではなく推定
- 「推定」札を付ける

## 吹き出し案

「ここはまだ確定ではなく、発掘途中の破片として扱います…」
