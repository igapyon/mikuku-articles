# テーブル設計は古い地図になる

## 主題

DB を使う既存業務システムでは、テーブル設計が古い地図として効く。物理名と論理名の対応が、業務意味の理解を助ける。

## 見たい情報

- テーブル名
- カラム名
- 型 / 桁 / NULL / 主キー / インデックス
- テーブル論理名
- カラム論理名
- 項目説明
- コード値の意味

## 図の流れ

```text
物理名だけ
  ↓ 読みにくい
論理名・項目説明
  ↓
業務上の意味
  ↓
ソースコード解析が進む
```

## 注意点

- DB コメントにあれば取得しやすい
- なければ定義書や項目定義書を探す
- 古くても現行スキーマと照合できる
- 多少ずれていても、ないより助かる

## 吹き出し案

「古い地図でも、業務意味を読む手がかりになります…」
