# コメント、ログ、メモも遺物になる

## 主題

正式な設計書でなくても、コメント、変更履歴、リリースノート、運用メモ、Excel は文脈復元の材料になる。

## 遺物と役割

| 遺物 | 役割 |
| --- | --- |
| コメント | 当時の判断や迷い |
| Git ログ | いつ、何が、なぜ変わったか |
| リリースノート | 業務上の出来事との接点 |
| 運用メモ | ソースコード外の前提 |
| Excel | 現場の理解や手作業の正本 |

## 変化

```text
ソースコードだけの説明
  ↓ 有識者知識・メモを追加
業務上の意味が乗る
  ↓
仕様メモの質が変わる
```

## 注意点

- コメントは常に正しいとは限らない
- 断片でも AI agent の文脈復元に効く
- 公式ではない資料も効くことがある

## 吹き出し案

「小さなコメントも、未来の仕様復元では手がかりになります…」
