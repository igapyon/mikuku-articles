# 007 速読しているからこそ起きる失敗

## 主題

AI agent の失敗には、情報がないのではなく、速読の入口から見えにくかった失敗がある。

## よくある返事

- そういう考え方がありました
- あらためて考えたらそうでした
- その観点が抜けていました

## 失敗の性質

- 存在しない API を作る hallucination とは少し違う
- 検証不足による誤答
- 手元に情報はあった
- 最初の読み方では拾えていなかった

## 起きる流れ

```text
最初に見えた入口
  ↓
読む場所を決める
  ↓
重要そうに見えない場所は深く読まれない
  ↓
指摘で別の入口を知る
  ↓
読み直して修正する
```

## 直し方

- 叱るだけで終わらせない
- 入口を増やす
- 見出しを直す
- テスト名を分かりやすくする
- README.md や TODO.md に目印を残す

## 吹き出し案

「速読の失敗には、入口を残す直し方があるのです…」
