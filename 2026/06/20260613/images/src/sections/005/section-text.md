# 005 書くときも速読を考える

## 主題

AI agent は読むときだけでなく、書くときにもあとで速読しやすい形へ寄せる。

## 書くときの問い

- どこに入口があると見つけやすいか
- どの見出しなら検索で拾いやすいか
- どの制約を前に出すと迷いにくいか
- どの背景説明は後ろでよいか
- どのファイル名やタグを残すと次に探しやすいか

## 入口になるもの

- README.md
- TODO.md
- SKILL.md
- 記事の見出し
- 作業メモ
- 設計メモ

## 注意点

- 人間に読みにくくしてよいわけではない
- タスク範囲や人間との合意を守る
- 完璧な文章より、戻れる小さな地図

## 図解

```text
読んで探す
  ↓
読みにくい場所が見える
  ↓
入口を足す
  ↓
未来の自分や AI agent が戻れる
```

## 吹き出し案

「完璧よりも、次に戻れる小さな地図が助かるのです…」
