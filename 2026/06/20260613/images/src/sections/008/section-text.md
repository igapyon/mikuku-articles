# 008 おわりに

## 主題

AI agent と開発するなら、コードもプロンプトも記事も、速読で見える形にしておく。

## まとめ

- AI agent は全部を同じ深さでは読まない
- 原則として、まず速読する
- 書くときも、あとで読みやすい形へ寄ることがある
- 人間にも AI agent にも、最初に見る入口が必要

## 速読で見える開発資産

- 迷わず入口を見つけられる
- 検索で必要な制約へたどり着ける
- 変更後にどこを確認すればよいか分かる

## 図解

```text
コード / プロンプト / 記事
  ↓
速読で見える入口
  ↓
必要な制約へ到達
  ↓
確認場所が分かる
  ↓
失敗しにくい開発スタイル
```

## 吹き出し案

「速読で見える形にしておくと、一緒に開発しやすいのです…」
