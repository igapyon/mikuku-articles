# グラレコ制作用整理テキスト

## 1. 大テーマ

**GPT-5.5 と Qwen3-Thinking では、Agent Skills の見え方が違う**

- GPT-5.5: 粗い文脈を「場」として受け取り、実務的に丸める
- Qwen3-Thinking: 粗い文脈も真面目に推論対象にする
- 結論: 知識は分けず、読み方と進め方を調整する

## 2. 中心メッセージ

```text
共通コンテンツ型 Skill
        ↓
蒸留層で意味の地図にする
        ↓
Qwen 用の駆動型 Skill を fork
        ↓
同じ知識を、モデルごとに安定して使う
```

## 3. 主要な対比

| 観点 | GPT-5.5 | Qwen3-Thinking |
| --- | --- | --- |
| 文脈の扱い | 粗さを補って丸める | 粗さも深く拾う |
| Agent Skills の見え方 | 命令書 + 場 | 推論材料 |
| 強み | 意図補完、実務的調整 | 真面目な深い推論 |
| 注意点 | 粗さに気づきにくい | 矛盾や古さに影響されやすい |
| 必要な工夫 | 共通文脈の整理 | 蒸留層 + 駆動層 fork |

## 4. Agent Skills の二分類

### コンテンツ型 Agent Skills

- 背景
- 価値観
- 判断基準
- 用語
- 文体
- 事例
- 設計思想
- 過去の経緯

役割: **作業空間に文脈を置く**

### 駆動型 Agent Skills

- 読む順番
- 確認タイミング
- 作業の刻み方
- エラー時の戻り方
- 出力形式

役割: **作業の進め方を補う**

## 5. 分けるもの / 分けないもの

```text
分けない:
  共通コンテンツ型 Agent Skills
  = 知識の正本

分けてもよい:
  駆動型 Agent Skills
  = モデル別の読み方、確認方法、進め方
```

## 6. 蒸留層の役割

**粗い素材を、意味の地図へ変換する層**

- 古い資料
- 粗い Markdown
- 実装観察
- DB 定義
- 画面仕様
- API
- 未確認事項

これらを整理して:

- 現行仕様として見えること
- 古い資料との差分
- まだ断定できないこと
- 元資料へ戻るべき場合

を先に見える形にする。

## 7. 運用サイクル

```text
複数モデルで使う
  ↓
Qwen3 が粗さを露出する
  ↓
矛盾・古さ・混在を見つける
  ↓
共通コンテンツを蒸留する
  ↓
Agent Skills の品質が上がる
  ↓
GPT-5.5 も Qwen3 も安定する
```

## 8. グラレコで強調する絵

- 左: GPT-5.5 の「場として読む」雰囲気
- 右: Qwen3-Thinking の「深く拾う」雰囲気
- 中央: 共通コンテンツ型 Skill
- 下: 蒸留層「意味の地図」
- さらに下: Qwen 用の駆動型 Skill fork
- 右端: みくくが「知識は分けず、読み方を整えます…」と説明

## 9. まとめの一言

**知識の正本は増やさない。共通コンテンツを育て、モデルごとの駆動層で安定させる。**
