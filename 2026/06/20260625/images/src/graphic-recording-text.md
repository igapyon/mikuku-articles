# グラレコ制作用整理テキスト

## 1. 大テーマ

- GOAL運用する Agent Skill
- AIエージェントの作業状態を Markdown に外へ出す
- 会話だけに頼らず、プロジェクト側に現在地を残す
- 小さな状態管理を Agent Skill として再利用する

## 2. 前回から今回への流れ

```text
前回: 3つの Markdown
  GOAL.md / TODO.md / DECISIONS.md
        ↓
運用してみる
        ↓
毎回、手順を思い出すのが少し辛い
        ↓
今回: Agent Skill に切り出す
        ↓
HANDOFF.md を追加して 4ファイル運用へ
```

## 3. 4つの状態管理ファイル

| ファイル | 役割 | グラレコ表現 |
| --- | --- | --- |
| `GOAL.md` | 目的、完了条件、止まる条件 | 目的地の旗 |
| `TODO.md` | 現在タスク、次の作業、詰まり | 現在地の地図 |
| `DECISIONS.md` | 判断理由、見送った案 | 分岐点のメモ |
| `HANDOFF.md` | 次の人間/AIへの再開メモ | 小さな封筒 |

キーワード:

- 目的
- 現在地
- 判断理由
- 引き継ぎ
- 軽く保つ
- 長い作業ログにしない

## 4. Setup と Resume の分担

```text
Setup
  状態管理ファイルを作る
  既存ファイルに合わせて整える
  TODO.md は上書きしない

Resume
  まず読む
  git status / README / TODO / GOAL / DECISIONS / HANDOFF
  目的・次タスク・詰まり・判断・引き継ぎを整理
```

対比:

- 作る前に読む
- 新規セットアップ / 作業再開
- ファイル作成 / 状態理解
- 長い初期プロンプト / 短い再開指示

## 5. 既存 TODO.md を壊さない

中心メッセージ:

- 人間の棚を勝手に並べ替えない
- AIエージェント用の小さな引き出しだけ足す

```text
既存 TODO.md
  ├─ 人間用メモ
  ├─ リリース残作業
  ├─ 記事管理
  └─ ## AI Agent Current Tasks
       ├─ current task
       ├─ blocker
       └─ retry log
```

## 6. front matter は設定ではなくヒント

- 公式設定ファイルではない
- `purpose` / `read_when` / `update_when`
- 読み手が用途を判断しやすくする札
- AIにも人間にも効く小さなラベル

対比:

- 魔法の設定ではない
- でも、読むべきタイミングを示す
- 強制ではなく、判断材料

## 7. Agent Skill にした意味

```text
毎回の会話に長い運用プロンプトを貼る
        ↓
SKILL.md を入口にする
        ↓
references / templates を必要に応じて読む
        ↓
短い指示で状態管理へ戻れる
```

キーワード:

- プロンプトの外に足場を置く
- Context Engineering
- 作業時に参照できる場所
- 運用ルールを Skill 側へ
- 小さな実例

## 8. 運用は育つ

```text
運用する
  ↓
足りないところが見える
  ↓
Skill / templates / references を調整
  ↓
次回の作業が少し安定
  ↓
また運用する
```

注意点:

- `GOAL.md` が古いと古い目的に引っ張られる
- `TODO.md` が更新されないと現在地がずれる
- `DECISIONS.md` に書きすぎると作業ログになる
- `HANDOFF.md` が古いと次の人やAIが迷う

## 9. グラレコで見せたい構図

- 左: 前回の3ファイルから出発
- 中央: `igapyon-agent-state-management` の Skill 化
- 右: 4ファイル運用と Setup / Resume
- 下: 運用しながら育つ循環
- 右端: みくくがやさしく説明

吹き出し案:

- 「あ、あの…状態は会話の外にも置けるのです…」
- 「まず読む、が大事かもしれません…」
- 「TODO.md は壊さず、小さな引き出しだけ足します…」
- 「小さな Markdown と小さな Skill で、戻ってこられる目印になります…」

## 10. まとめ

- 状態管理はファイル名を決めたら終わりではない
- 軽く、でも更新されることが大事
- AIエージェントが読みやすく、人間も確認できることが大事
- 小さな Markdown ファイルと小さな Agent Skill が、長い作業の迷いを減らす
