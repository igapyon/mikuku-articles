# グラレコ制作用整理テキスト

## 1. 大テーマ

- アジャイル入門
- スクラムやツールの前に、まず宣言へ戻る
- 2001年の現場実践から生まれた「価値の置き方」
- 4つの価値を基準点にして、12原則と現代実践を見直す

## 2. 中央メッセージ

```text
アジャイル = 自由にやることではない
        = 迷ったときに何をより大事にするか
```

- 原典へ戻る
- 4つの価値を読む
- 12原則で日々の開発へつなぐ
- スクラム、XP、カンバン、ツールを「価値を支える具体化」として見る

## 3. 歴史の入口

- 2001年2月
- Utah Snowbird
- 17人のソフトウェア開発関係者
- XP / Scrum / DSDM / Crystal / FDD / Pragmatic Programming など
- 背景: 重くなりすぎた文書主導プロセスへの違和感

```text
現場実践
  ↓
共通する違和感
  ↓
大事にしたい価値
  ↓
Agile Manifesto
```

## 4. 4つの価値

| より重く見るもの | ただし捨てないもの | 図解キーワード |
|---|---|---|
| 人と対話 | プロセスやツール | 会話、意図確認、調整 |
| 動くソフトウェア | 分厚い文書 | 実物、確認、学び |
| 顧客との協調 | 契約交渉 | 一緒に確かめる |
| 変化への対応 | 計画に従うこと | 仮説、地図、調整 |

```text
プロセス・ツール・文書・契約・計画
      価値はある
          ↓
でも、迷ったらより大事にするものがある
          ↓
対話 / 動くもの / 協調 / 変化対応
```

## 5. 12原則の見取り図

- 早く継続的に価値を届ける
- 変化する要求を受け入れる
- 短い間隔で動くものを届ける
- ビジネス側と開発者が日々協力する
- 意欲ある人を信頼し支援する
- 直接対話を重視する
- 動くソフトウェアを進捗の尺度にする
- 持続可能なペース
- 技術的卓越性とよい設計
- シンプルさ
- 自己組織的なチーム
- 定期的なふりかえりと調整

```text
4つの価値 = 判断の軸
12原則   = 日々の開発での扱い方
```

## 6. 現代実践への橋

| 現代の言葉 | つながる原則 | 見え方 |
|---|---|---|
| CI/CD | Frequent delivery / Technical excellence | 短い間隔で届ける仕組み |
| 継続的デリバリー | Early and continuous value | 早く継続的に価値を届ける |
| DevOps / BizDevOps | Daily collaboration | 分断せず、動くものを中心に協力 |
| バックログ見直し | Welcome changing requirements | 学びに応じて優先順位を変える |
| レトロスペクティブ | Regular reflection | 定期的にふりかえり調整 |
| 自己組織化チーム | Self-organizing teams | 現場で判断できる状態 |
| テスト自動化 / リファクタリング | Technical excellence | 継続的に品質を支える |
| 小さく作って確かめる | Working software / Simplicity | 動くものとシンプルさ |

## 7. 広がりと注意

- 原典はソフトウェア開発のための宣言
- ただし価値観は他領域にも応用された
- プロダクトマネジメント
- 組織運営
- マーケティング
- 教育
- 業務改善

```text
すべてを最初に決めきれない
関係者との対話が必要
試してみないと価値がわからない
計画を持ちながら変える必要がある
```

## 8. 対比

| 誤解 | この記事での見方 |
|---|---|
| アジャイル = スクラム | スクラムは価値観を具体化する方法のひとつ |
| アジャイル = ツール運用 | ツールより前に価値判断がある |
| アジャイル = 文書を書かない | 文書にも価値はあるが、動くものを重く見る |
| アジャイル = 計画しない | 計画は仮説として持ち、変化に合わせて調整する |
| アジャイル = 自由 | 何を優先するかの判断軸 |
