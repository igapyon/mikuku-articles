# Copilot Studio は、データ作りが重要になる

## 主題

Copilot Studio は作る入口は簡単に見えるが、良いエージェントにはデータ整備が必要。

## 仕組み

```text
knowledge source
  +
generative answers
  ↓
質問に対して根拠を探して答える
```

## データ整備

- FAQ形式
- 手順形式
- 具体的な見出し
- 版・有効日
- 意味のある列名
- 古い資料を退避
- 引用元タイトル整理

## ファイル形式の見方

```text
Microsoft 365 運用
  -> Office形式が自然

AI向け知識構造
  -> Markdown も強い
```

## 吹き出し

「入口はやさしくても、裏側のデータ整備が効いてくるのです…」
