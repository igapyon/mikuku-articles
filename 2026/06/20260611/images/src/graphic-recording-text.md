# グラレコ制作用整理テキスト

## 1. 大テーマ

- 生成AI時代のアジャイル
- 4つの価値は消えない
- 価値の「場所」「速度」「役割分担」「判断軸」が変わる
- 生成AIはアジャイルを不要にしない
- アジャイルの価値を近い距離で試させる

## 2. 中央メッセージ

```text
アジャイル宣言
  ↓ 生成AIで読み直す
4つの価値が新しい現れ方をする
  ↓
動くもの・対話・協調・変化が近くなる
```

## 3. 4つの価値の読み替え

| 価値 | Before 生成AI | After 生成AI |
|---|---|---|
| 人と対話 | 人と人が話し、専門家が後で形にする | 人と人の対話に生成AIも加わり、その場で理解・構造化・試作を支える |
| 動くソフトウェア | 後で動くものを見て確認する | すぐに、または目の前で動くものを作りながら確認する |
| 顧客との協調 | 顧客が話し、専門家が具体化する | 顧客と専門家が、生成AIも使って意図・コスト・価値を一緒に確かめる |
| 変化への対応 | 変更には大きな作業コストがある | 変更を小さく試しつつ、生成AIが変える社会や業務にも対応する |

## 4. 図解の骨格

```text
          生成AI時代のアジャイル
                  |
      --------------------------------
      |          |          |        |
    対話       動くもの     協調      変化
      |          |          |        |
  AIも参加   その場で試作  一緒に確認  小さく試す
      \          |          |        /
       -------- 人間の判断が中心 -------
```

## 5. 価値ごとの視覚キーワード

### 人と対話

- 会議室に増えたもう一つの席
- AIが要約・構造化・論点整理
- 人間の意図合わせは前より重要
- アイコン: 人、吹き出し、AIの席、メモ

### 動くソフトウェア

- 後で見るものから、今ここで触るものへ
- たたき台、画面、試作品
- 違和感や期待を触りながら見つける
- アイコン: 画面、再生ボタン、プロトタイプ、手元の小さなアプリ

### 顧客との協調

- 顧客は要望を伝える人から、AIと試して表現する人へ
- 専門家は作る人から、試せる環境と検証軸を作る人へ
- コスト、モデル選択、推論回数も一緒に確認
- アイコン: 顧客、専門家、AI、コストメーター

### 変化への対応

- AIで変更や試作は速くなる
- でも大きすぎる変化は重い
- 変化を小さく分けて試す
- AIが変える社会・業務・期待にも対応
- アイコン: 分割された矢印、循環、計画表、変化の波

## 6. Before / After の大きな対比

```text
Before:
話す → 持ち帰る → 作る → 次回レビュー

After:
話す → その場で構造化 → すぐ試作 → 触って確認 → 小さく修正
```

## 7. 役割分担

| 要素 | 役割 |
|---|---|
| 人間 | 意図、責任、判断、価値の確認 |
| 顧客 | 困りごと、期待、違和感の表現 |
| 専門家 | 構造化、検証、本番品質、リスク判断 |
| 生成AI | 要約、試作、観点提示、作り直しの支援 |
| 動くソフトウェア | 対話を進める共通の対象 |

## 8. 循環構造

```text
対話
 ↓
生成AIで構造化
 ↓
すぐ試作
 ↓
顧客と触る
 ↓
違和感・価値・コストを確認
 ↓
小さく変える
 ↓
次の対話へ
```

## 9. 注意ポイント

- 生成AIの答えを正解扱いしない
- 動いただけでは十分な証拠ではない
- 意味があるか、安全に扱えるか、運用できるかを見る
- 速く変えられるからこそ、変え方を小さく整える

## 10. まとめの一文

```text
生成AIはアジャイルを置き換えない。
アジャイルの4つの価値を、
もっと近い距離で、速く、小さく試せるようにする。
```
