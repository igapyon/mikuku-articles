# グラレコ制作用整理テキスト

## 1. テーマ

AIエージェントの状態管理を、まずは3つのMarkdownから小さく始める。

```text
会話だけに置く記憶
  ↓ 揮発しやすい
プロジェクト内のMarkdown
  ↓ 残る・読める・差分を追える
AIと人間の共有メモ
```

## 2. 問題意識

- 目的が曖昧になる
- 判断理由が見えなくなる
- 同じ案や失敗を繰り返す
- 現在地が分からなくなる
- TODO消化だけで本来の目的からずれる

### 見せたい対比

| 会話だけ | Markdownに外部化 |
| --- | --- |
| 文脈が長くなると埋もれる | プロジェクト側に残る |
| ツールを移ると切れやすい | Codex / Copilot / Claude Code で読める |
| 判断理由が薄れる | 決定理由を再利用できる |
| 人間も思い出しづらい | 人間も確認できる |

## 3. 最小構成

中央に3つのファイルを三角形で配置する。

```text
              GOAL.md
        終わり方・止まり方
             /       \
            /         \
 TODO.md  --------  DECISIONS.md
 現在地          判断理由
```

## 4. 3ファイルの役割

| ファイル | 役割 | 入れるもの | 入れすぎないもの |
| --- | --- | --- | --- |
| GOAL.md | 終わり方を決める | Objective / Done / Stop | 細かいタスク一覧 |
| TODO.md | 現在地を保つ | Tasks / Blockers / Retry Log | 長い作業日誌 |
| DECISIONS.md | 判断理由を残す | 理由 / 影響 / 見送った案 | 全思考ログ |

## 5. GOAL.md

- 完了条件を見る場所
- タスクリストではない
- Stop条件も置く
- 同じ失敗が3回なら止まる

短い言葉:

- 終わり方
- 止まり方
- 完了基準
- スコープ確認

## 6. TODO.md

- AI Agent Current Tasks
- Blockers
- Retry Log
- 次の一手が分かる粒度
- 既存TODOがある場合はセクション追加で住み分け

短い言葉:

- 現在地
- 足元の道しるべ
- 詰まりを記録
- ループ検知

## 7. DECISIONS.md

- なぜその方針にしたか
- どの案を見送ったか
- 何を優先したか
- 何が後続タスクになったか

短い言葉:

- 判断理由
- 巻き戻り防止
- ループ回避
- 決めた背景

## 8. ファイル自身にヒントを埋め込む

front matterを「公式設定」ではなく「読む人とAIへのヒント」として使う。

```text
purpose
read_when
update_when
```

狙い:

- 毎回長いプロンプトを貼らない
- ファイルを読んだAIが用途を判断しやすい
- 人間にも運用意図が伝わる

## 9. Markdownにする理由

- 特定ツールに閉じない
- AIエージェントを持ち替えても読める
- 人間も読める
- Gitで差分を追える
- レビューできる

図解:

```text
Codex
GitHub Copilot agent  →  GOAL / TODO / DECISIONS  →  共有メモ
Claude Code
人間
```

## 10. Prompt Engineering から Context Engineering へ

良い指示を書くことだけでなく、AIが迷わない状態を維持する。

```text
Prompt Engineering
  「どうお願いするか」
        ↓
Context Engineering
  「迷わない状態をどう保つか」
```

## 11. 運用サイクル

```text
作業開始
  ↓ 3ファイルを読む
目的・現在地・判断理由を確認
  ↓
作業する
  ↓
TODO / DECISIONS を更新
  ↓
GOAL の Done / Stop で確認
  ↓
次回も再開しやすい
```

## 12. これから試す検証観点

- 目的を見失いにくくなるか
- 同じ判断のやり直しが減るか
- 作業再開時に現在地へ戻りやすいか
- ファイル更新の負担が重すぎないか
- 3ファイルで足りない場面はどこか

## 13. まとめメッセージ

「小さく始める。でも曖昧に始めない。」

`GOAL.md`、`TODO.md`、`DECISIONS.md` は、AIエージェントと人間が同じ作業状態を見失わないための小さな足場。
