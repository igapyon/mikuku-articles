# Root index maintenance

## Purpose

The repository root has a generated index file:

- `index.json`

This file is generated by `miku-indexgen`. Do not edit it by hand.

## When To Update

Update the root index file whenever repository content changes in a way that should
be discoverable by agents or readers.

Common triggers:

- A new Note article directory is added under `2026/`.
- Article Markdown files are added, renamed, or removed.
- Image-generation prompt Markdown files are added or changed.
- Documentation under `docs/` is added or changed.
- The existing generated index file is missing or stale.

## Command

Run `miku-indexgen` from the repository root:

```sh
java -jar /Users/igapyon/.codex/skills/igapyon-miku-indexgen/runtime/miku-indexgen-1.4.4.jar \
  --input-directory . \
  --output-directory . \
  --title "mikuku-articles"
```

This generates or refreshes:

```text
index.json
```

## Expected Scope

The generated index scans supported text files under the repository root. In this
repository, that mainly means:

- Article Markdown files under `2026/`.
- Image prompt and section source Markdown files under article image directories.
- Documentation Markdown files under `docs/`.
- Existing JSON files that match the generator's supported extension list.

Binary image files such as `.jpg` and `.png` are not listed as ordinary text entries
in the generated JSON index.

## Verification

After running the command, check:

```sh
git status --short
sed -n '1,40p' index.json
```

Before committing, confirm:

- `index.json` was generated by `miku-indexgen`.
- No manual edits were made to the generated index file.
- The index title is `mikuku-articles`.
- The generated file is committed together with the content changes that made the
  index stale.

## Notes

If `index.json` already exists and contains generation metadata, a refresh command
may also be used:

```sh
java -jar /Users/igapyon/.codex/skills/igapyon-miku-indexgen/runtime/miku-indexgen-1.4.4.jar \
  --refresh-index index.json
```

When in doubt, regenerate from the repository root without `--markdown` so only
`index.json` is maintained.
